# 1주차 학습 문서

1주차의 목표는 문자열, 배열, 기본 구현 문제를 통해 코딩테스트의 가장 기초적인 처리 능력을 만드는 것이다.

이 주차는 어렵고 화려한 알고리즘을 배우는 주간이 아니다.  
오히려 아래 능력을 만드는 주간이다.

- 문제를 끝까지 읽는 습관
- 예외 케이스를 먼저 떠올리는 습관
- 문자열/배열을 코드로 정확하게 다루는 능력
- 구현 실수 없이 정답까지 가는 힘
- `왜 이 풀이가 맞는지` 짧게 설명하는 습관

## 1주차 목표

- 프로그래머스 Level 1 문자열/배열 문제를 안정적으로 푼다.
- Java의 문자열/배열 기본 조작에 익숙해진다.
- 한 문제를 풀고 나서 코드 주석으로 핵심을 짧게 정리한다.
- 쉬운 문제라도 시간 복잡도와 예외 케이스를 생각하는 습관을 만든다.
- SQL `SELECT` 기본 문법을 익히고, 아주 쉬운 조회 문제를 직접 풀기 시작한다.

## 1주차 운영 방식

1주차는 알고리즘이 메인이고, SQL은 매일 20~30분 정도 짧게 병행한다.

- 알고리즘: 하루 2문제
- SQL: 하루 0~1문제
- SQL 목표: `SELECT`, `FROM`, `WHERE`, `ORDER BY`, `LIMIT`에 익숙해지기

중요한 점:
- SQL 때문에 알고리즘 문제 수를 줄이지 않는다.
- SQL은 "깊게"보다 "매일 조금씩" 가져간다.
- 1주차 SQL은 어려운 JOIN이 아니라, 조회문에 익숙해지는 단계다.

## 이번 주에 공부할 알고리즘 / 개념

1주차는 엄밀히 말해 "고급 알고리즘"보다는 코딩테스트 기본기를 공부하는 주차다.  
아래 항목들을 확실히 다루는 것이 핵심이다.

### 1. 문자열 처리

핵심 개념:
- 문자열 길이 확인
- 특정 위치 문자 접근
- 부분 문자열 자르기
- 문자 단위 순회
- 문자열 정렬
- 문자열 조건 검사
- 문자열 누적 조합

Java에서 꼭 익힐 것:
- `length()`
- `charAt()`
- `substring()`
- `toCharArray()`
- `toLowerCase()`, `toUpperCase()`
- `StringBuilder`
- `String.valueOf()`

왜 중요한가:
- 프로그래머스 Level 1은 문자열 구현 문제가 매우 자주 나온다.
- 문자열 문제는 난이도보다 구현 실수 때문에 틀리는 경우가 많다.
- 이후 해시, 스택, 완전탐색으로 넘어가도 문자열 처리는 계속 나온다.

### 2. 배열 처리

핵심 개념:
- 배열 순회
- 인덱스 기반 접근
- 최댓값/최솟값 탐색
- 부분 배열 다루기
- 원소 정렬
- 중복 제거 기초

Java에서 꼭 익힐 것:
- `arr.length`
- 기본 `for` 문
- 향상된 `for` 문
- `Arrays.sort()`
- `ArrayList<Integer>`

왜 중요한가:
- 대부분의 코테 문제는 결국 배열 또는 리스트를 다루는 문제로 바뀐다.
- 배열 처리 실수가 줄어야 이후 투 포인터, 정렬, 완전탐색이 편해진다.

### 3. 정렬 기초

핵심 개념:
- 오름차순 정렬
- 내림차순 정렬 발상
- 문자/문자열 정렬
- 정렬 후 원하는 값 추출

Java에서 꼭 익힐 것:
- `Arrays.sort()`
- `Collections.sort()`
- 문자열 배열 정렬

왜 중요한가:
- 코테에서는 정렬만 해도 문제가 매우 단순해지는 경우가 많다.
- 1주차부터 정렬을 가볍게 섞어두면 이후 정렬 주차가 훨씬 쉬워진다.

### 4. 시간 복잡도 감각

핵심 개념:
- 이중 반복문이 필요한지 확인
- 문자열을 반복해서 더하면 느려질 수 있다는 점
- 입력 크기에 비해 풀이가 과한지 판단

이번 주에서는 아래 정도만 판단하면 충분하다.
- 문자열/배열 한 번 순회: `O(N)`
- 정렬: `O(N log N)`
- 2중 반복문: `O(N^2)`

왜 중요한가:
- 아직 어려운 최적화는 필요 없지만, "이 풀이가 너무 무거운가?"를 생각하는 습관은 지금부터 들어야 한다.

## 문제를 풀 때 매번 체크할 질문

문제를 읽고 바로 아래 질문을 떠올리면 좋다.

1. 입력 크기가 큰가?
2. 문자열 문제인가, 배열 문제인가?
3. 정렬을 하면 쉬워지는가?
4. 예외 케이스가 무엇인가?
5. 구현 전에 손으로 예시를 돌려봤는가?

## 1주차에서 자주 나오는 실수

### 1. 문제를 다 읽기 전에 구현 시작

증상:
- 요구사항 하나를 놓친다.
- 예외 케이스를 빼먹는다.

해결:
- 구현 전에 입출력 예를 손으로 먼저 확인한다.

### 2. 문자열 더하기를 반복해서 사용

증상:
- 코드가 길어지고 느려진다.

해결:
- 누적 문자열은 기본적으로 `StringBuilder`를 먼저 떠올린다.

### 3. 인덱스 경계 실수

증상:
- `IndexOutOfBoundsException`
- 부분 문자열 범위 오류

해결:
- `substring`의 끝 인덱스는 포함되지 않음을 항상 체크한다.
- `for` 문의 시작과 끝을 먼저 주석으로 써본다.

### 4. 정렬 후 자료형 처리 실수

증상:
- 문자 정렬과 숫자 정렬을 혼동한다.
- 정렬 후 다시 문자열/숫자로 만드는 과정에서 꼬인다.

해결:
- "정렬 대상이 char인지 String인지 int인지" 먼저 명확히 한다.

### 5. 예외 케이스 미처리

예시:
- 공백 포함 문자열
- 길이가 1인 문자열
- 짝수 길이 / 홀수 길이
- 중복 원소 존재

해결:
- 구현 전에 최소 3개 예외 케이스를 먼저 적어본다.

## 1주차 종료 기준

1주차가 끝났을 때 아래가 되면 잘 진행 중인 것이다.

- Level 1 문자열/배열 문제를 보고 겁먹지 않는다.
- 문자열, 배열, 정렬 기본 구현을 혼자 처리할 수 있다.
- 한 문제를 푼 뒤 핵심 아이디어를 코드 주석으로 3~4줄 남길 수 있다.
- 쉬운 문제를 풀 때 실수는 있어도, 무엇이 문제였는지 설명할 수 있다.
- SQL에서 `SELECT`, `FROM`, `WHERE`, `ORDER BY`를 보고 의미를 설명할 수 있다.
- 아주 쉬운 SELECT 문제는 정답을 보지 않고 풀 수 있다.
- 문자열 정렬과 기본 Comparator 풀이를 설명할 수 있다.

## 1주차가 끝난 뒤 다음으로 넘어가기 전에 확인할 것

- `StringBuilder`를 자연스럽게 쓰는가?
- 배열 정렬과 문자열 정렬을 구분하는가?
- 부분 문자열과 인덱스 처리가 익숙한가?
- 쉬운 문제를 20~40분 안에 마무리할 수 있는가?
- SQL에서 기본 조회문을 스스로 쓸 수 있는가?
- ORDER BY와 WHERE를 함께 쓸 수 있는가?

이 질문들에 어느 정도 "예"라고 답할 수 있으면 2주차 해시로 넘어가면 된다.
