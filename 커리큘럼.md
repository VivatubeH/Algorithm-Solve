# 12주 코딩테스트 커리큘럼

프로그래머스 기준으로 코딩테스트 합격 가능한 기초 실력을 만드는 것을 목표로 하는 12주 커리큘럼이다.

핵심 방향은 아래와 같다.

- 자주 나오는 유형을 먼저 익힌다.
- 이론을 길게 보지 않고 바로 문제에 적용한다.
- 많이 푸는 것보다 다시 풀 수 있게 만드는 데 집중한다.
- 코드 상단 또는 내부 주석으로 발상, 시간 복잡도, 실수 포인트를 짧게 남긴다.
- 알고리즘은 메인 트랙으로, SQL은 매일 조금씩 병행한다.

## 운영 원칙

- 권장 학습일: 주 6일
- 권장 휴식/복습일: 주 1일
- 하루 권장 알고리즘 문제 수: 2문제
- 하루 권장 SQL 문제 수: 0~1문제
- 주당 권장 알고리즘 문제 수: 12문제 내외
- 주당 권장 SQL 문제 수: 3~5문제
- 12주 총 권장 알고리즘 문제 수: 130문제 전후
- 12주 총 권장 SQL 문제 수: 40문제 전후

## 알고리즘 + SQL 병행 원칙

- 알고리즘이 메인이다. SQL 때문에 알고리즘 학습량이 줄어들면 안 된다.
- SQL은 별도 날을 만드는 대신, 매일 20~30분 정도 짧게 병행한다.
- 알고리즘 문제를 푼 뒤 집중력이 조금 내려가는 구간에 SQL 1문제를 붙이는 방식이 가장 좋다.
- 알고리즘 실전 주간에는 SQL도 복잡한 문제보다 복습 위주로 간다.
- 초반 주차가 너무 쉽게 느껴지면, 같은 주 안에서 다음 Day를 당겨서 진행해도 된다.
- 다만 주차 자체를 무조건 앞당기기보다, 해당 주의 핵심 개념을 다 확인했는지 먼저 본다.

권장 비중:
- 알고리즘 70~80%
- SQL 20~30%

## 진도 가속 규칙

초반 Level 1 문제는 현재 실력에 따라 예상보다 빨리 끝날 수 있다.  
이 경우는 계획이 무너진 것이 아니라, 기초 구현력이 이미 어느 정도 있다는 신호다.

아래 원칙으로 조정한다.

1. 같은 날 Day 1, Day 2를 함께 끝내도 괜찮다.
2. 남는 시간은 다음 Day를 당겨서 진행한다.
3. 단, 주차 핵심 주제는 건너뛰지 않는다.
4. 쉬운 문제를 빨리 끝낸 만큼 복습일을 늘리거나, 쉬운 Level 2를 조금 일찍 섞는다.
5. SQL도 같은 방식으로 조금씩 앞당길 수 있지만, 알고리즘 진도를 침범하지는 않는다.

## 문제 풀이 규칙

1. 한 문제를 40분 이상 붙잡지 않는다.
2. 40분 이상 막히면 힌트 또는 풀이를 본다.
3. 풀이를 본 문제는 24시간 안에 다시 혼자 푼다.
4. 문제를 풀고 나면 코드 주석으로 아래 4가지만 남긴다.

```java
/*
 * 알고리즘:
 * 발상:
 * 시간복잡도:
 * 놓친 포인트:
 */
```

## 주간 공통 루틴

### 월요일 ~ 금요일
- 이론 20~30분
- 알고리즘 문제 2문제 풀이
- SQL 문제 0~1문제 풀이
- 코드 주석으로 짧은 회고 작성

### 토요일
- 이번 주 핵심 알고리즘 유형 복습
- 틀렸던 알고리즘 문제 2~3개 다시 풀기
- SQL 오답 1~2문제 복습

### 일요일
- 휴식 또는 가벼운 복습
- 2주에 한 번씩 실전처럼 시간 재고 2~3문제 풀기

## 하루 권장 템플릿

1. 알고리즘 이론 또는 전날 오답 복습 20분
2. 알고리즘 문제 2문제 풀이 60~100분
3. 알고리즘 코드 주석 회고 10분
4. SQL 문제 1문제 또는 SQL 개념 복습 20~30분

핵심은 SQL을 "별도 큰 공부"로 두지 않는 것이다.  
매일 짧게 붙여서 감을 유지하는 방식이 가장 효율적이다.

## SQL 병행 순서

알고리즘 12주와 함께 SQL은 아래 순서로 병행한다.

1. SELECT
2. SUM, MAX, MIN
3. GROUP BY
4. IS NULL
5. String, Date
6. JOIN

이 순서는 프로그래머스 SQL 고득점 Kit의 기본 흐름과도 잘 맞고, 쉬운 것부터 점점 코테형으로 올라가기 좋다.

## 1단계: 구현/자료구조 기초 다지기

### 1주차: 문자열 + 배열 + 구현

목표:
- 문제를 끝까지 읽고 구현하는 습관 만들기
- 문자열/배열 관련 기본 메서드에 익숙해지기
- SQL SELECT 기본 문법 감 잡기

학습 주제:
- 문자열
- 배열
- 기본 구현
- 시간 복잡도 기초

집중할 포인트:
- `charAt`, `substring`, `toCharArray`
- `StringBuilder`
- `Arrays.sort`
- 예외 케이스 먼저 보기

추천 플랫폼 구간:
- 기초 문제
- 입문 문제
- Level 1 문자열/배열 문제

SQL 병행:
- SELECT
- `SELECT`, `FROM`, `WHERE`, `ORDER BY`, `LIMIT`
- SQL 문제 3개 정도

### 2주차: 해시

목표:
- 해시를 써야 하는 문제를 빠르게 구분하기
- SQL 집계 함수 기본기 익히기

학습 주제:
- `HashMap`
- `HashSet`
- 빈도수 세기
- 중복 제거
- 존재 여부 확인

집중할 포인트:
- 정렬 없이 풀 수 있는지 먼저 보기
- key/value 의미를 명확히 두기
- `getOrDefault`에 익숙해지기

추천 플랫폼 구간:
- 입문 문제
- 알고리즘 고득점 Kit `해시`

SQL 병행:
- SUM, MAX, MIN
- `COUNT`, `SUM`, `MAX`, `MIN`, `DISTINCT`
- SQL 문제 3~4개

### 3주차: 스택 + 큐

목표:
- 순서 관리가 필요한 문제를 자료구조로 자연스럽게 연결하기
- 묶어서 계산하는 SQL 감각 익히기

학습 주제:
- 스택
- 큐
- 덱 기초
- 괄호 문제
- 순서 처리 문제

집중할 포인트:
- `push/pop/peek`
- `offer/poll/peek`
- 조건 분기보다 자료구조로 해결하는 감각

추천 플랫폼 구간:
- 입문 문제
- 알고리즘 고득점 Kit `스택/큐`

SQL 병행:
- GROUP BY 기초
- `GROUP BY`, `COUNT`, `ORDER BY`
- SQL 문제 3~4개

### 4주차: 정렬 + 구현 응용

목표:
- 정렬 후 푸는 사고방식 익히기
- SQL NULL 처리에 익숙해지기

학습 주제:
- 숫자 정렬
- 문자열 정렬
- 조건 기반 정렬
- 정렬 후 비교/선택

집중할 포인트:
- 정렬이 필요한 문제인지 먼저 판단
- Comparator 작성
- 정렬 후 문제 구조가 단순해지는지 확인

추천 플랫폼 구간:
- 입문 문제
- 알고리즘 고득점 Kit `정렬`

SQL 병행:
- IS NULL
- `IS NULL`, `IS NOT NULL`, `IFNULL`, `COALESCE`
- SQL 문제 3개 정도

## 2단계: 완전탐색과 시간 단축 감각 만들기

### 5주차: 완전탐색 + 시뮬레이션

목표:
- 가능한 모든 경우를 체크해도 되는지 판단하기
- SQL 문자열/날짜 처리 기본기 익히기

학습 주제:
- 브루트포스
- 순회
- 시뮬레이션
- 조건 구현

집중할 포인트:
- 입력 크기 보고 완전탐색 가능 여부 판단
- 문제를 쪼개서 구현
- 실수 없는 순차 처리

추천 플랫폼 구간:
- 입문 문제
- 알고리즘 고득점 Kit `완전탐색`

SQL 병행:
- String, Date 기초
- `LIKE`, `DATE_FORMAT`, 문자열 함수
- SQL 문제 3~4개

### 6주차: 투 포인터 + 슬라이딩 윈도우

목표:
- O(N^2) 풀이를 O(N)으로 줄이는 감각 익히기
- SQL String/Date를 실전형으로 확장

학습 주제:
- 같은 방향 투 포인터
- 양 끝 투 포인터
- 슬라이딩 윈도우

집중할 포인트:
- 포인터가 한 방향으로만 움직이는지
- 구간 합/개수 갱신 방식
- 정렬 필요 여부

추천 플랫폼 구간:
- Level 2 쉬운 문제
- 투 포인터 관련 문제 모음

SQL 병행:
- String, Date 심화
- 날짜 조건 필터링, 문자열 조건 검색
- SQL 문제 3~4개

### 7주차: 누적합 + 구간합

목표:
- 구간 합 문제를 빠르게 처리하는 패턴 익히기
- SQL JOIN 입문

학습 주제:
- 1차원 누적합
- 구간 합
- 빈도 누적

집중할 포인트:
- 미리 계산해 두고 빠르게 조회한다는 발상
- 인덱스 처리 실수 줄이기
- `prefix[r] - prefix[l - 1]` 패턴 익히기

추천 플랫폼 구간:
- 구간합/누적합 유형 문제

SQL 병행:
- JOIN 기초
- `INNER JOIN`, `LEFT JOIN`, 조인 조건 읽기
- SQL 문제 3개 정도

### 8주차: 이분 탐색 + 매개변수 탐색

목표:
- 값을 찾는 탐색과 정답을 찾는 탐색을 구분하기
- JOIN 문제 대응력 높이기

학습 주제:
- 이분 탐색
- lower bound / upper bound 감각
- 매개변수 탐색

집중할 포인트:
- 정렬 여부 확인
- 범위 설정
- `이 조건을 만족하는가?` 형태로 바꾸기

추천 플랫폼 구간:
- 이분 탐색 관련 문제
- 알고리즘 고득점 Kit 중 중급 진입 문제

SQL 병행:
- JOIN 심화
- 집계 + JOIN, 정렬 + JOIN
- SQL 문제 3~4개

## 3단계: Level 2 핵심 유형 적응

### 9주차: 우선순위 큐 + 그리디

목표:
- 최솟값/최댓값을 반복적으로 선택하는 문제에 익숙해지기
- SQL 유형별 혼합 문제 적응 시작

학습 주제:
- `PriorityQueue`
- 정렬 기반 그리디
- 선택 기준 세우기

집중할 포인트:
- 지금의 최선 선택이 전체에서도 유효한지 생각하기
- 우선순위 큐가 필요한 상황 파악

추천 플랫폼 구간:
- 알고리즘 고득점 Kit `힙`
- 쉬운 그리디 문제

SQL 병행:
- SELECT + GROUP BY + JOIN 혼합
- SQL 문제 3개 정도

### 10주차: DFS + BFS

목표:
- 탐색 문제 기본 틀을 익히고 방문 처리에 익숙해지기
- JOIN과 GROUP BY를 함께 쓰는 문제에 익숙해지기

학습 주제:
- DFS
- BFS
- 그래프/격자 탐색 기초

집중할 포인트:
- 방문 배열 처리
- 재귀와 반복 구현 차이
- 큐/스택을 활용한 탐색 흐름

추천 플랫폼 구간:
- 알고리즘 고득점 Kit `DFS/BFS`

SQL 병행:
- JOIN + GROUP BY 혼합
- SQL 문제 3개 정도

### 11주차: 기본 DP

목표:
- 작은 문제의 답을 쌓아 큰 문제를 푸는 감각 만들기
- SQL 약점 유형 보완

학습 주제:
- 1차원 DP
- 점화식
- 메모이제이션

집중할 포인트:
- 상태 정의
- 점화식 만들기
- 초기값 설정

추천 플랫폼 구간:
- 쉬운 DP 문제
- Level 2 입문 DP

SQL 병행:
- 틀렸던 파트 재풀이
- JOIN 또는 String/Date 취약 파트 보강
- SQL 문제 3~4개

## 4단계: 실전 적응과 약점 보완

### 12주차: 실전 모의 + 약점 보완

목표:
- 제한 시간 안에 문제를 고르고 푸는 흐름 만들기
- 알고리즘과 SQL 모두 실전 감각 정리

학습 주제:
- 스킬체크
- 혼합 유형 문제
- 오답 복습

집중할 포인트:
- 문제를 읽고 쉬운 순서대로 푸는 전략
- 오래 걸리는 문제를 버리는 기준
- 실수 유형 정리

추천 진행:
- 주 2회 60~90분 실전 세트
- 기존 오답 중 중요 문제 다시 풀기
- 약한 유형 1개만 집중 보완

SQL 병행:
- SQL 혼합 세트 복습
- 자주 틀리는 문법만 압축 정리
- SQL 문제 2~3개 가볍게 복습

## 12주 동안 꼭 체크할 것

### 1. 매주 점검 질문
- 내가 이번 주에 가장 많이 틀린 이유는 무엇인가?
- 시간 초과, 구현 실수, 자료구조 선택 미스 중 무엇이 많았는가?

- Level 1 문제를 안정적으로 푸는가?

- 특정 유형만 보면 겁나는 구간이 있는가?

### 3. 실력 판단 기준
- 4주차 종료: Level 1 안정화
- 8주차 종료: 자주 나오는 유형 대부분 경험
- 12주차 종료: 쉬운 Level 2까지 실전 대응 가능
- SQL은 12주차 종료 시 SELECT, GROUP BY, JOIN 기본 문제를 스스로 해결할 수 있으면 충분하다.

## 추천 공부 방식

- 한 번에 오래 하는 것보다 매일 꾸준히 하는 것이 훨씬 중요하다.
- 풀이를 외우지 말고, 왜 그 자료구조를 선택했는지 설명하는 연습을 해야 한다.
- 같은 문제를 다시 풀었을 때 더 빨라졌는지가 중요하다.

## 내가 도와줄 수 있는 방식

- 문제를 풀기 전: 유형 분류와 접근 방향 점검
- 문제를 푼 후: 코드/주석 기준 피드백
- 주차 종료 후: 약점 유형 분석
- 다음 주 시작 전: 문제 세트 추천
