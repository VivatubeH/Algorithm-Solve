# 📂 [Algo] 백준 17232 - 생명 게임

> 날짜: 2026-01-24 | 난이도: Gold 4

### 1. 핵심 아이디어 💡
* **알고리즘**: 누적합(Prefix Sum)
* **핵심 발상**: 시간 복잡도를 어떻게 줄일지?
* 1. 매번 주위 생명 검사 : `O((2K + 1)^2 * N * M * T)` - 시간 초과
* 2. 생명 수 누적합 생성 이후 구간합 이용 : `O(N * M + T * N * M + N * M)` - 시간 내 가능. 

### 2. 코드 리뷰 & 로직 ⭐
* **주의점**: 생명 게임 결과는 새로운 바둑판에 기록이 되야함. ( 다음 생명게임에 영향을 주면 안됨. )
* **로직**
* 1. 최초의 바둑판을 토대로 생명 개수를 누적합 배열에 기록한다.
* 2. T초에 대해 생명 게임을 진행한다.
* - 누적합 배열을 이용해 구간합을 통해 주위 생명 개수를 구함.
* - 주위 생명 개수를 토대로 현재 칸의 생명 변화를 기록함.
* - 현재 칸에 기록된 생명을 토대로 생명 누적합 배열을 새로 생성함.

### 3. 트러블슈팅 & 배운 점 🚩
- **누적합 사용** : 누적합을 이용해서 시간을 어떻게 단축시킬 것인지
- **범위 처리** : 범위를 벗어나는 경우는 사전에 제외해서 index 오류를 방지한다.
- **독립 시행** : 이전칸의 수행이 다음 칸에 수행에 영향을 미치지 않도록, 기존은 절대 변경하지 않는다.
- **예외 케이스 처리** : 생명이 없는데, 탄생이 아닌 케이스도 반드시 기록은 되어야 한다.
