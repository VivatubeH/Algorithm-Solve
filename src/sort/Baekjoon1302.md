# 📂 [Algorithm] 백준 1302 - 베스트셀러

> **관련 링크**: https://www.acmicpc.net/problem/1302
> **난이도**: Silver 4
> **학습 날짜**: 2026-01-21

---

## 1. 문제 해석 🔍
* **목표**: 하루 중 가장 많이 팔린 책의 제목 출력하기
* **입력 값**: 하루 동안 팔린 책의 개수 N ( 1 ~ 1000권 )의 이름 (50자 이하의 알파벳 소문자)
* **제약 사항**: 시간 제한 2초, 메모리 제한 128메가
* **핵심 포인트**: 가장 많이 팔린 책이 여러권이면 사전순으로 가장 앞인 책 출력

## 2. 해결 전략 (아이디어) 💡
* **사용 알고리즘**: 정렬
* **풀이 프로세스**:
    1. Map 객체에 책 제목과 팔린 권수를 담는다. getOrDefault로 이미 Map에 존재하는 책이든 아니든 1권씩 늘어나게 한다.
    2. 정렬을 위해 List에 담는데, Map은 Collection을 상속받지 않아서 내부의 Set을 담기위해 Entry를 쓴다.
    3. Set 내부의 Entry의 Key, Value를 기반으로 정렬 후 가장 많이 팔린 책인 0번 책을 출력한다.
* **핵심 아이디어**: 팔린 권수가 포함되어야 하므로 map을 쓴다.

## 3. 복잡도 분석 ⏳
* **시간 복잡도**: O(N logN * 문자열 길이) - 책의 제목을 사전순 정렬해야 함
* **공간 복잡도**: 빈도가 필요하므로 Map 객체 1000개

## 4. 핵심 코드 리뷰 ⭐
* **정렬 로직**: Key가 책 제목, Value가 팔린 권수를 이용해 삼항 연산자 + 람다식으로 1줄에 나타냈다.

## 5. 트러블슈팅 / 느낀 점 🚩
* **실수했던 점**: List에 Map을 담으려는 실수를 했다.
* **배운 점**
- ArrayList 생성자의 인자로는 Collection 인터페이스의 구현체만 가능하다. 고로 Map은 불가.
- 단, Map 내부 인터페이스인 Entry의 entrySet() 메서드는 Set을 반환하게 된다.
- Set은 Collection 인터페이스의 구현체이므로 인자로 전달이 가능하다.
