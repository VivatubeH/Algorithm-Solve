# 📂 [Algorithm] 백준 2910 - 빈도 정렬

> **관련 링크**: https://www.acmicpc.net/problem/2910
> **난이도**: Silver 3
> **학습 날짜**: 2026-01-21

---

## 1. 문제 해석 🔍
* **목표**: 수열이 주어질 때 빈도가 많이 나오는 수가 앞에 있도록 수열 출력하기
* **입력 값**: 메시지의 숫자 N개 ( 1 ~ 1000), 등장하는 숫자 C ( 1에서 10억 )
* **제약 사항**: 시간 제한 1초, 메모리 제한 128MB
* **핵심 포인트**: 등장 빈도가 같으면 -> 먼저 나온 게 앞에 있어야 함. 

## 2. 해결 전략 (아이디어) 💡
* **사용 알고리즘**: 정렬
* **풀이 프로세스**:
    1. 정렬의 기준이 될 변수가 3개라서, 자체 클래스를 만들었다.
    2. 자체 클래스를 Map의 Key인 숫자에 대한 Value로서 담는다.
    3. ArrayList에 Entry.entrySet()을 이용해서 Set을 담고 정렬 기준을 통해 정렬한다.
    4. 정렬된 Key인 숫자마다 자체 클래스의 count만큼 출력을 해주면 된다.
* **핵심 아이디어**: Map에는 Key, Value 2개만 담을 수 있으니 자체 클래스 생성

## 3. 복잡도 분석 ⏳
* **시간 복잡도**: O(N logN) - 수를 정렬
* **공간 복잡도**: 자체 클래스 1000개 (int 변수 2개), Map 1000개

## 4. 핵심 코드 리뷰 ⭐
* **Frequencyt 클래스**: 정렬을 위해 2개의 정보를 담는 클래스를 생성했다.

## 5. 트러블슈팅 / 느낀 점 🚩
* **개선 방향**: 람다식이 좀 길어져서, 그냥 내부에서 {}로 쓰는 게 이번 문제에선 더 가독성 있었을 것 같다. 
* **배운 점**: Entry를 추가하기 위해서는, util 패키지가 아닌 util.Map까지 import 해야 함
- 추가로 다음에 이런 문제가 나오면, 자바 자체의 LinkedHashMap을 썼어도 됐을 것 같다.
- LinkedHashMap은 HashMap과는 달리 넣은 순서를 기억하는 특성이 있다.
