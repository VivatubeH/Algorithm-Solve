# 📂 [Algorithm] 백준 7785 - 회사에 있는 사람

> **관련 링크**: https://www.acmicpc.net/problem/7785
> **난이도**: Silver 5
> **학습 날짜**: 2026-01-21

---

## 1. 문제 해석 🔍
* **목표**: 회사에 있는 사람을 사전 역순으로 출력
* **입력 값** : 출입 기록의 수 ( 2 ~ 100만)만큼 이름이 들어옴, 이름은 최대 5글자
* **제약 사항**: 동명이인은 없음, 대소문자 다를 시 다른 이름
* **핵심 포인트**: 현재 회사에 있는 사람만 + 사전순 역순 출력

## 2. 해결 전략 (아이디어) 💡
* **사용 알고리즘**: 정렬
* **풀이 프로세스**:
    1. Set 자료구조에 사원의 이름을 출근일 때 저장, 퇴근일 때 삭제한다.
    2. 정렬을 위해 List 자료구조로 복사한다.
    3. List를 문자열의 정렬인 compareTo를 통해 정렬 후 출력한다.
* **핵심 아이디어**: 회사에 있는 사람을 어떻게 추려낼지? 

## 3. 복잡도 분석 ⏳
* **시간 복잡도**: O(N logN) : 최대 N명에 대한 정렬이 필요
* **공간 복잡도**: Set으로 String 100만개

## 4. 핵심 코드 리뷰 ⭐
* **출퇴근 체크**: Set에 있으면 출근, 없으면 퇴근이거나 들어온 적 없거나

## 5. 트러블슈팅 / 느낀 점 🚩
* **실수했던 점**: Map을 처음에 생각했는데, 출근이면 Set에 남아있기 때문에 불필요하다.
* **개선 방향**: Set의 contains를 처음에 썼는데, add, remove의 특성 상 불필요하다.
* **배운 점**
- Set의 add, remove의 동작 기전 ( 가능할 때는 수행 후 true 반환, 불가능 시 false 반환)
- 인덱스가 있는 List 인터페이스의 구현체만이 sort 가능.
