# 📂 [Algorithm] 백준 10814 - 나이순 정렬

> **관련 링크**: https://www.acmicpc.net/problem/10814
> **난이도**: Silver 5
> **학습 날짜**: 2026-01-19

---

## 1. 문제 해석 🔍
* **목표**: 나이와 이름을 정렬해서 출력
* **입력 값**: 회원의 수 (1명에서 10만명)만큼 나이, 이름 
* **제약 사항**: 시간 제한 3초, 메모리 제한 256MB
* **핵심 포인트**: 입력이 가입한 순서대로 주어짐. (판단의 근거)

## 2. 해결 전략 (아이디어) 💡
* **사용 알고리즘**: 정렬
* **풀이 프로세스**:
    1. 정렬을 위해 데이터를 담을 클래스를 정의한다.
    2. 입력받은 데이터를 인덱스를 가입 순서로 활용해서 객체로 만든 뒤 리스트에 담는다.
    3. 모두 리스트에 담은 뒤, 람다식을 이용해 새로운 정렬 기준을 부여해 정렬한다.
* **핵심 아이디어**: 입력 순을 가입 순으로? 인덱스를 쓰자.

## 3. 복잡도 분석 ⏳
* **시간 복잡도**: O(N logN) - 이름, 나이 정렬 
* **공간 복잡도**: 사용자 정의 객체 최대 10만개에 int 2개, String 1개 

## 4. 핵심 코드 리뷰 ⭐
* **Member 클래스**: 가입 순서가 필수조건이므로 대소비교를 위해 필드를 만들었다.

## 5. 트러블슈팅 / 느낀 점 🚩
* **개선 방향**
* - Collections.sort는 Timsort라서 같은 위상 시 stable 하다고 했다.
* - 그렇다는 건, list에 들어간 순서가 유지되기 때문에 가입순을 체크할 필요는 없다.
* **배운 점**: 람다식을 쓰는 것에 조금 익숙해졌다.
