# 📂 [Algo] 백준 2003 - 수들의 합 2

> 날짜: 2026-01-28 | 난이도: Silver 4

### 1. 핵심 아이디어 💡
* **알고리즘**: 투 포인터(Two Pointer)
* **핵심 발상**: 

#### i, j를 선택한 이후 i부터 j까지 합 구하기
- i와 j를 선택하기 O(N^2)
- i부터 j까지 합 구하기 O(N)
- 총 시간 복잡도 O(N^3) -> 시간 초과

#### 누적합 이용하기
- 전처리 : i부터 j까지의 누적합 구하기 O(N)
- i와 j를 선택하기 O(N^2) * 구간합 조회 O(1)
- 총 시간 복잡도 O(N^2) - 시간 초과 (문제 시간 제한 0.5초)

#### i, j 선택 시 투포인터 + 누적합 이용
- 전처리 : i부터 j까지의 누적합 구하기 O(N)
- i와 j를 선택 시 투포인터 : O(N)
- 총 시간 복잡도 O(N) * O(1) = O(N) 시간 내에 가능

#### i, j까지 투포인터 + 합은 변수 하나로 관리
- 전처리 : 필요 X
- i, j 선택 시 투포인터 : O(N)
- 총 시간 복잡도 O(N) = 시간 내에 가능.

### 2. 코드 리뷰 & 로직 ⭐
* **주의점**: 수열의 모든 요소가 자연수, 우측으로 갈 수록 합이 커짐이 보장된다. ( 단조성 )
* **코드 로직** : 
* - 합이 M보다 작다 -> right 포인터를 우측으로 이동시킨다.
* - 합이 M보다 크거나 같다. -> right 포인터 이동을 종료하고, left를 이동시켜 구간을 좁힌다.
- 합이 M일 때만 체크한다.

### 3. 트러블슈팅 & 배운 점 🚩
- right : 항상 다음에 더할 인덱스를 가리킨다.
- while에서 항상 인덱스 체크를 먼저 쓰는 습관을 들인다.
- if 체크는 while문과 분리해서 while문 이후 멈춘 자리에서 판단한다.
- right - left는 구간의 길이가 된다.
