# 📂 [Algorithm] 백준 2817 - ALPS식 투표

> **관련 링크**: https://www.acmicpc.net/problem/2817
> **난이도**: Silver 3
> **학습 날짜**: 2026-01-19

---

## 1. 문제 해석 🔍
* **목표**: 5% 이상 득표한 스태프의 이름과 칩 개수를 계산해서 사전순 출력
* **입력 값**: 참가자 수 X는 1명에서 250만명, 스태프 수는 0명에서 최대 10명
* **제약 사항**: 시간 제한 1초, 메모리 제한 128MB
* **핵심 포인트**: 참가자가 득표를 하지 않을 수도 있다!! ( 고로, 득표수를 합해서 득표율을 구해야함 )

## 2. 해결 전략 (아이디어) 💡
* **사용 알고리즘**: 정렬 
* **풀이 프로세스**:
    1. 참가자수를 기반으로 득표율을 구한다.
    2. 등장 스태프를 체크하기 위해, 알파벳 인덱스를 기반으로 한 배열을 생성한다.
    3. 득표율 5% 이상인 스태프에 대해서만 점수 집합을 생성하고, **5% 이상인 스태프는 별도로 체크한다.**
    4. 이후 점수집합에서 점수 기준으로 정렬 후 상위 14명의 인덱스를 토대로 체크한다.
    5. 이후 출력시에 득표율이 5% 이상인 사람에 대해 빈도를 출력한다. 자동 사전순 출력
  * **핵심 아이디어**: 자체 클래스 생성 및 출력 기준을 위해 boolean 배열 생성

## 3. 복잡도 분석 ⏳
* **시간 복잡도**: O(N logN) - N명에 대해 점수 집합 정렬
* **공간 복잡도**: int 배열 N칸, char 배열 N칸, 점수집합을 다루기 위한 클래스 객체 최대 140개

## 4. 트러블슈팅 / 느낀 점 🚩
* **실수했던 점**: 득표율 산정 기준을 처음에 잘못 파악했고, 출력 조건을 제대로 파악하지 못했다.
* **개선 방향**: 이름이 알파벳이라는 조건을 활용하는 법 ( 알파벳 인덱스 활용 )
