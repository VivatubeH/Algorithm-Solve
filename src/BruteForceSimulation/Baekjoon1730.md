# 📂 [Algorithm] 백준 1730 - 판화

> **관련 링크**: https://www.acmicpc.net/problem/1730
> **난이도**: Silver 4
> **학습 날짜**: 2026-01-18

---

## 1. 문제 해석 🔍
* **목표**: 로봇 팔의 이동 후 조각판의 흔적 출력
* **입력 값**: 목판 크기 N은 2와 10사이, 로봇팔 움직임은 최대 250 길이 문자열
* **제약 사항**: 시간 제한 2초, 메모리 제한 128 MB
* **핵심 포인트**: 작동 중 바깥으로 나갈 명령어는 아예 무시됨.

## 2. 해결 전략 (아이디어) 💡
* **사용 알고리즘**: 브루트포스 중 시뮬레이션
* **풀이 프로세스**:
    1. 로봇팔의 움직임을 항상 기억하기 위한 변수를 메서드 내에서 유지한다.
    2. 로봇팔이 움직일 때, 유효하지 않은 움직임이면 갱신하지 않고, 좌표도 유지한다.
    3. 유효한 움직임일시, 수직이면 수직 체크, 수평이면 수평 체크를 한다.
* **핵심 아이디어**: char 변수에 기록 시 코드가 복잡, boolean으로 체크 시 로직이 간단함

## 3. 복잡도 분석 ⏳
* **시간 복잡도**: O(문자열 길이) + O(N^2)
* **공간 복잡도**: String 변수 하나, boolean 200칸이면 충분 

## 4. 핵심 코드 리뷰 ⭐
* **moveRobot**: 실제 문자열 길이만큼 내부에서 로봇팔을 움직이며 갱신한다.
* **로봇팔 내의 움직임**: nrow, ncol을 초기화 시키지 않아서 비정상적인 무빙이 된 적이 있음.

## 5. 트러블슈팅 / 느낀 점 🚩
* **실수했던 점**: 다음번 이동이 유효하지 않으면, nrow, ncol을 둘 다 원위치 해야함. 
* **개선 방향**: 처음 char 배열에서 boolean 배열로 바꾸었다.
* **배운 점**: 한 방향 이동이어도 양 방향 갱신이 모두 필요함. ( 좌표에서 )
