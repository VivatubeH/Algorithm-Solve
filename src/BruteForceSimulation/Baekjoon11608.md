- 유형 : 문자열
- 회문인 수

##### 문제 해석
1. 회문인 수 : 왼쪽부터 읽든, 오른쪽으로 읽든 같은 수
2. 단, 255도 16진수로 표현시 FF로 회문인 수가 된다.
3. B진법으로 회문이 되는 경우가 있는지 알려주는 프로그램 작성하기
4. 테스트 케이스는 T개이고, 각 데이터는 64 ~ 100만 이하의 정수

##### 풀이
1. B진법으로 표현해서 회문인지 체크하려면 2~64진법까지 체크가 필요함.
- 하나라도 true이면 회문인 수가 된다.
- 다 돌았을 때 회문인 수를 찾지 못하면 false

##### 포인트
1. 시간 복잡도를 생각할 때,
- 2~64진법까지에 대해서 변환 ( 63개의 데이터에 대해 )
- 특정 수에 대한 B진법 변환에 쓰이는 연산 ( 1회성 변환 )
- B진법 변환이 회문인지 체크하는 연산 ( 문자열의 양끝에서부터 좁혀가며 검사 )
- 시간 내에 충분히 들어올 데이터 크기이다.

2. 진법 계산처럼 수식을 해야하는 경우, 순서에 특히 유의해야 한다.
- 이 문제를 풀면서, 나머지를 기록하는 과정보다 나누는 과정을 먼저 한 경우 오류가 발생.
- 나머지를 먼저 기록하고, 이후에 나눠야 됨. ( 구하고자 하는 나머지가 뭘지를 잘 생각해보기 )

3. 'Z' 이후의 문자가 char로 들어가는 경우가 생기는데, 이 문제에선 통과되었으나
- 만약 문자 자체가 중요하다면 지양하는 게 좋고
- 이 문제 같은 경우는, 단순히 int로서 배열에 담고 앞뒤를 비교하는 법도 좋음.

4. 단순히 String만 쓰면, "1162"처럼 되어서 회문 판단이 안됨.
