- 유형 : 배열
- 두 수의 합

##### 문제 해석
1. 입력되는 수의 크기는 1개에서 10만개 사이.
2. 수열에 포함되는 수는 1과 100만 사이의 자연수
3. 자연수 x가 주어졌을 때, 수열의 두 수를 더해서 x가 되는 쌍의 개수 구하기

##### 풀이
1. O(N^2) 사용 시 최악의 경우 시간 초과.
2. boolean 100만개 사용 시 -> 1000개에 1kb, 1000 x 1000 = 100만개에 1mb
- 메모리 여유는 있음.
- 입력 시에 해당 수와 더해서 X를 만들 수 있는 수가 boolean에 존재하는지 체크.
- 존재하는 경우 count 해줌. 
- 이후, 해당 수도 boolean에 존재한다고 체크
3. 핵심: 더해서 X가 되는 수가 체크하는지 체크 -> count -> 해당 수도 체크
- 예를 들어, 5가 입력으로 주어질 때 8이 있는지 체크
- 8이 없으면 count가 올라가지 않고, 이후 5가 존재한다고 체크됨.
- 추후 8이 들어올 경우, 5가 이미 체크되어 있으니 count 되고 8도 체크됨.
- 즉 매칭되는 수가 들어와있을 때만, 중복없이 1회만 체크가 가능하다.

##### 포인트
1. 핵심은 이중 반복을 피하면서, 최적의 시간 복잡도를 찾는 것이다.
2. 배열을 사용할 때는 반드시 인덱스가 의미하는 바가 무엇인지를 생각한다.
